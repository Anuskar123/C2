Final Practice Questions

What are the four fundamental principles of Object-Oriented Programming?

The four pillars of OOP are:
1. Encapsulation - Bundling data and methods, hiding implementation details
2. Abstraction - Focusing on essential features while ignoring irrelevant details
3. Inheritance - Creating new classes from existing ones, promoting code reuse
4. Polymorphism - Ability of objects to take many forms (compile-time and runtime)

Find the memory leak in this code and explain how to fix it:

class Shape {
public:
    virtual double area() = 0;
};

class Circle : public Shape {
private:
    double radius;
public:
    Circle(double r) { radius = r; }
    double area() { return 3.14 * radius * radius; }
};

int main() {
    Shape* s = new Circle(5);
    cout << s->area() << endl;
    delete s;
    return 0;
}

The issue is that the base class Shape doesn't have a virtual destructor. When deleting a derived object through a base pointer, this causes undefined behavior and potential memory leaks.

Fix: Add virtual ~Shape() {} to the base class to ensure proper cleanup of derived objects.

What is the difference between pass-by-value, pass-by-reference, and pass-by-pointer?

Pass-by-value: Creates a copy, changes don't affect original
Pass-by-reference: Passes memory address using &, changes affect original
Pass-by-pointer: Passes memory address using *, requires null checks

Reference is generally safer and cleaner than pointer for non-null parameters.

Write a complete Stack class with proper encapsulation and error handling.

Here's a complete Stack implementation:

class Stack {
private:
    int* arr;
    int top;
    int capacity;
public:
    Stack(int size = 100) {
        capacity = size;
        arr = new int[capacity];
        top = -1;
    }
    
    ~Stack() {
        delete[] arr;
    }
    
    void push(int value) {
        if (top >= capacity - 1) {
            throw "Stack overflow";
        }
        arr[++top] = value;
    }
    
    int pop() {
        if (isEmpty()) {
            throw "Stack underflow";
        }
        return arr[top--];
    }
    
    int peek() const {
        if (isEmpty()) {
            throw "Stack is empty";
        }
        return arr[top];
    }
    
    bool isEmpty() const {
        return top == -1;
    }
    
    bool isFull() const {
        return top >= capacity - 1;
    }
};

Explain the concept of virtual functions and when to use them.

Virtual functions enable runtime polymorphism. They're declared with virtual keyword in base class and can be overridden in derived classes. The correct function is called based on actual object type, not pointer/reference type.

Use when:
â€¢ You have inheritance hierarchy
â€¢ You want derived classes to provide different implementations
â€¢ You're using base class pointers/references to derived objects

What are some common mistakes beginners make with pointers?

Common pointer mistakes:
1. Dangling pointers - Using deleted memory
2. Memory leaks - Forgetting to delete allocated memory
3. Null pointer dereference - Accessing null pointers
4. Pointer arithmetic errors - Incorrect array indexing
5. Double deletion - Deleting the same memory twice
6. Uninitialized pointers - Using garbage addresses

Design a simple Student class with proper encapsulation and validation.

Here's a well-designed Student class:

class Student {
private:
    string name;
    int id;
    vector<int> grades;
    
public:
    Student(string n, int i) : name(n), id(i) {
        if (id <= 0) throw "Invalid student ID";
    }
    
    void addGrade(int grade) {
        if (grade < 0 || grade > 100) {
            throw "Grade must be between 0 and 100";
        }
        grades.push_back(grade);
    }
    
    double getAverage() const {
        if (grades.empty()) return 0.0;
        int sum = 0;
        for(int grade : grades) sum += grade;
        return static_cast<double>(sum) / grades.size();
    }
    
    string getName() const { return name; }
    int getId() const { return id; }
    const vector<int>& getGrades() const { return grades; }
};

Explain the concept of templates and provide a practical example.

Templates enable generic programming by allowing functions and classes to work with different data types. The compiler generates specific code for each type used.

// Function template
template <typename T>
T maximum(T a, T b) {
    return (a > b) ? a : b;
}

// Class template
template <typename T>
class Container {
private:
    T data;
public:
    Container(T val) : data(val) {}
    T getData() const { return data; }
};

// Usage
int main() {
    cout << maximum(5, 10) << endl;           // int
    cout << maximum(3.14, 2.71) << endl;     // double
    
    Container<int> c1(42);
    Container<string> c2("Hello");
}

Write a function to reverse a string without using standard library functions.

Here's a manual string reversal function:

string reverseString(string str) {
    int left = 0;
    int right = str.length() - 1;
    
    while (left < right) {
        // Swap characters
        char temp = str[left];
        str[left] = str[right];
        str[right] = temp;
        
        left++;
        right--;
    }
    
    return str;
}

int main() {
    string original = "Hello World";
    string reversed = reverseString(original);
    cout << reversed << endl;  // Output: dlroW olleH
    return 0;
}

What are smart pointers and why should you use them?

Smart pointers are wrapper classes that automatically manage memory. They prevent memory leaks by automatically deleting objects when they go out of scope.

Types:
â€¢ unique_ptr - Exclusive ownership
â€¢ shared_ptr - Shared ownership with reference counting
â€¢ weak_ptr - Non-owning reference to shared_ptr

Benefits: Automatic cleanup, exception safety, prevention of memory leaks.

Design a simple file I/O class for reading and writing text files.

Here's a simple file I/O utility class:

class FileHandler {
public:
    static vector<string> readLines(const string& filename) {
        vector<string> lines;
        ifstream file(filename);
        
        if (!file.is_open()) {
            throw "Could not open file: " + filename;
        }
        
        string line;
        while (getline(file, line)) {
            lines.push_back(line);
        }
        
        file.close();
        return lines;
    }
    
    static void writeLines(const string& filename, 
                          const vector<string>& lines) {
        ofstream file(filename);
        
        if (!file.is_open()) {
            throw "Could not create file: " + filename;
        }
        
        for (const string& line : lines) {
            file << line << endl;
        }
        
        file.close();
    }
};

Write a program that demonstrates the use of inheritance and polymorphism.

Here's a complete example with shapes:

class Shape {
public:
    virtual double area() const = 0;
    virtual string getName() const = 0;
    virtual ~Shape() {}
};

class Circle : public Shape {
private:
    double radius;
public:
    Circle(double r) : radius(r) {}
    double area() const override {
        return 3.14159 * radius * radius;
    }
    string getName() const override {
        return "Circle";
    }
};

class Rectangle : public Shape {
private:
    double width, height;
public:
    Rectangle(double w, double h) : width(w), height(h) {}
    double area() const override {
        return width * height;
    }
    string getName() const override {
        return "Rectangle";
    }
};

int main() {
    vector<Shape*> shapes;
    shapes.push_back(new Circle(5));
    shapes.push_back(new Rectangle(4, 6));
    
    for (Shape* shape : shapes) {
        cout << shape->getName() << " area: " 
             << shape->area() << endl;
        delete shape;
    }
    
    return 0;
}
