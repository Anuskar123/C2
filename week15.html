Week 15: Inheritance and Polymorphism

Advanced Object-Oriented Programming Concepts

Week 15 Overview

This week covers advanced OOP concepts including inheritance, polymorphism, and virtual functions. These concepts allow for code reuse and dynamic behavior.

Key Topics:

  • Inheritance (single and multiple)
  • Base and derived classes
  • Virtual functions and polymorphism
  • Abstract classes and pure virtual functions
  • Function overriding
  • Access control in inheritance

Inheritance

Basic Inheritance Syntax

// Base class
class Vehicle {
protected:
    string brand;
    int year;

public:
    Vehicle(string b, int y) : brand(b), year(y) {}

    void displayInfo() {
        cout << "Brand: " << brand << ", Year: " << year << endl;
    }
};

// Derived class
class Car : public Vehicle {
private:
    int doors;

public:
    Car(string b, int y, int d) : Vehicle(b, y), doors(d) {}

    void displayCarInfo() {
        displayInfo(); // Call base class method
        cout << "Doors: " << doors << endl;
    }
};

Types of Inheritance

  • Public: Public members remain public, protected remain protected
  • Private: All inherited members become private
  • Protected: Public members become protected, protected remain protected

Polymorphism

Virtual Functions

class Shape {
public:
    virtual double area() const = 0; // Pure virtual function
    virtual void draw() const {
        cout << "Drawing a shape" << endl;
    }
};

class Circle : public Shape {
private:
    double radius;

public:
    Circle(double r) : radius(r) {}

    double area() const override {
        return 3.14159 * radius * radius;
    }

    void draw() const override {
        cout << "Drawing a circle" << endl;
    }
};

class Rectangle : public Shape {
private:
    double width, height;

public:
    Rectangle(double w, double h) : width(w), height(h) {}

    double area() const override {
        return width * height;
    }

    void draw() const override {
        cout << "Drawing a rectangle" << endl;
    }
};

Using Polymorphism

void displayArea(const Shape* shape) {
    cout << "Area: " << shape->area() << endl;
    shape->draw();
}

int main() {
    Circle circle(5.0);
    Rectangle rect(4.0, 6.0);

    displayArea(&circle);  // Calls Circle's methods
    displayArea(&rect);    // Calls Rectangle's methods

    return 0;
}

Practice Questions & Answers (20 Questions)

Question 1: Find the error in this inheritance code:

class Base {
private:
    int x;
public:
    Base(int val) : x(val) {}
};

class Derived : public Base {
public:
    Derived(int val) {
        // Constructor body
    }
};
Type: Find the Error

Error: Derived class constructor doesn't call base class constructor

Explanation: When a derived class is instantiated, the base class constructor must be called explicitly.

Fix: Derived(int val) : Base(val) { }

Output: A B ~B ~A

Explanation: Base constructor first, then derived. Destructors in reverse order.

Question 12: Write a function using dynamic_cast:

class Base {
public:
    virtual ~Base() {}
};

class Derived : public Base {
public:
    void specialFunc() { cout << "Special" << endl; }
};

void processObject(Base* obj) {
    // Write code to safely call specialFunc if obj is Derived
}
Type: Write a Function

Solution:

void processObject(Base* obj) {
    Derived* derived = dynamic_cast<Derived*>(obj);
    if (derived) {
        derived->specialFunc();
    }
}

Explanation: dynamic_cast safely converts base pointer to derived pointer, returns nullptr if conversion fails.

Question 13: Find the error in this diamond inheritance:

class A {

    int x;
};

class B : public A {};
class C : public A {};

class D : public B, public C {
public:
    void func() { cout << x << endl; }  // Error!
};
Type: Find the Error

Error: Ambiguous access to x

Explanation: Diamond inheritance creates two copies of A, making x ambiguous.

Fix: Use virtual inheritance: class B : virtual public A {};

Question 16: Write a function using polymorphism:

class Shape {
public:
    virtual double area() const = 0;
};

class Rectangle : public Shape {
private:
    double w, h;
public:
    Rectangle(double w, double h) : w(w), h(h) {}
    double area() const override { return w * h; }
};

class Circle : public Shape {
private:
    double r;
public:
    Circle(double r) : r(r) {}
    double area() const override { return 3.14 * r * r; }
};

// Write a function that works with any Shape
double totalArea(const vector<Shape*>& shapes) {
    // Your code here
}
Type: Write a Function

Solution:

double totalArea(const vector<Shape*>& shapes) {
    double total = 0;
    for (Shape* shape : shapes) {
        total += shape->area();
    }
    return total;
}

Explanation: Polymorphism allows treating different shapes uniformly through base class pointers.

Question 17: Find the error in this virtual function:

class Base {
public:
    virtual void func(int x) { cout << "Base: " << x << endl; }
};

class Derived : public Base {
public:
    void func(double x) { cout << "Derived: " << x << endl; }  // Error!
};
Type: Find the Error

Error: Not a valid override

Explanation: Parameter types differ (int vs double), so this is function hiding, not overriding.

Fix: Use same parameter type or use override keyword to catch this error.

Question 20: Write a function demonstrating late binding:

class Processor {
public:
    virtual void processData() = 0;
};

class TextProcessor : public Processor {
public:
    void processData() override {
        cout << "Processing text data" << endl;
    }
};

class ImageProcessor : public Processor {
public:
    void processData() override {
        cout << "Processing image data" << endl;
    }
};

// Write a function that demonstrates late binding
void processFile(Processor* proc) {
    // Your code here
}
Type: Write a Function

Solution:

void processFile(Processor* proc) {
    proc->processData();  // Late binding - method called at runtime
}

Explanation: Late binding occurs when virtual function calls are resolved at runtime based on actual object type.

Exercises

Exercise 1: Employee Hierarchy

Create a class hierarchy:

  • Base class: Employee (name, id, salary)
  • Derived classes: Manager, Developer, Intern
  • Each derived class should have specific attributes and overridden methods
class Employee {
protected:
    string name;
    int id;
    double salary;

public:
    Employee(string n, int i, double s) : name(n), id(i), salary(s) {}

    virtual double calculateBonus() const = 0;
    virtual void displayInfo() const {
        cout << "Name: " << name << ", ID: " << id << ", Salary: " << salary << endl;
    }
};

class Manager : public Employee {
private:
    int teamSize;

public:
    Manager(string n, int i, double s, int ts) : Employee(n, i, s), teamSize(ts) {}

    double calculateBonus() const override {
        return salary * 0.2; // 20% bonus
    }

    void displayInfo() const override {
        Employee::displayInfo();
        cout << "Team Size: " << teamSize << endl;
    }
};
