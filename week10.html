Week 10: Classes and Objects

Introduction to Object-Oriented Programming in C++

Week 10 Overview

This week introduces the fundamental concepts of Object-Oriented Programming (OOP) in C++. Think of OOP like building with LEGO blocks - each block (object) has its own properties and can do specific things, but they all follow the same basic design pattern (class).

üß† Think Like a Beginner

Real-World Analogy: A class is like a cookie cutter, and objects are the cookies you make with it. The cookie cutter defines the shape, but each cookie is a separate, edible instance!

Key Topics:

  • Class definition: The blueprint or template for creating objects
  • Creating objects: Making actual instances of your class (like stamping out cookies)
  • Constructors: Special setup functions that run when you create an object
  • Destructors: Cleanup functions that run when an object is destroyed
  • Member functions: Actions that objects can perform (like a car's "drive" or "stop" functions)
  • Access control: Deciding what data is private (secret) and what is public (shareable)
  • Data encapsulation: Protecting your data like keeping money in a wallet, not loose in your pocket

üí° Why OOP Matters

OOP helps you organize complex programs by breaking them into smaller, manageable pieces. It's like organizing your closet - instead of throwing everything in a pile, you use shelves, drawers, and boxes to keep things organized and easy to find.

Theoretical Concepts Made Simple

üéØ What is a Class?

Simple Explanation: A class is like a recipe for baking cookies. The recipe tells you what ingredients you need (data) and what steps to follow (functions), but it isn't a cookie itself. You use the recipe to make actual cookies (objects).

Technical Definition: A class is a user-defined data type that encapsulates data and functions into a single unit. It serves as a blueprint for creating objects.

üèóÔ∏è Class vs Object

üè† Class (The Blueprint)
  • Like a house plan
  • Defines structure and capabilities
  • Exists only once
  • Cannot be lived in directly
üè° Object (The Actual House)
  • Like a built house
  • Actual instance you can use
  • Can have multiple instances
  • You interact with objects

üîí Access Specifiers (Privacy Settings)

Think of it like social media privacy:

  • üåê Public: Like a public Facebook post - anyone can see and interact with it
  • üîê Private: Like your personal diary - only you (the class) can access it
  • üë®‚Äçüë©‚Äçüëß‚Äçüë¶ Protected: Like family photos - only you and your close relatives (derived classes) can see it

üß† Pro Tip for Beginners

Start with Public: When learning, make everything public first to avoid confusion. As you get comfortable, gradually add private members to protect your data.

Code Examples

Basic Class Definition

// Rectangle.h - Specification file
#ifndef RECTANGLE_H
#define RECTANGLE_H

class Rectangle
{
private:
    double width;
    double length;

public:
    // Constructor
    Rectangle(double w = 0.0, double len = 0.0)
    {
        width = w;
        length = len;
    }

    // Destructor
    ~Rectangle() {}

    // Member functions
    void setWidth(double w) { width = w; }
    void setLength(double len) { length = len; }
    double getWidth() const { return width; }
    double getLength() const { return length; }
    double getArea() const { return width * length; }
};

#endif

Using the Class

#include <iostream>
#include "Rectangle.h"
using namespace std;

int main()
{
    // Create objects
    Rectangle box1;           // Default constructor
    Rectangle box2(10.0, 5.0); // Parameterized constructor

    // Set dimensions
    box1.setWidth(8.0);
    box1.setLength(12.0);

    // Display results
    cout << "Box1 area: " << box1.getArea() << endl;
    cout << "Box2 area: " << box2.getArea() << endl;

    return 0;
}

Account Class Example

// Account.h
#ifndef ACCOUNT_H
#define ACCOUNT_H

class Account
{
private:
    double balance;
    int transactions;

public:
    Account(double initialBalance = 0.0)
    {
        balance = initialBalance;
        transactions = 0;
    }

    bool withdraw(double amount)
    {
        if (amount > balance)
            return false;
        balance -= amount;
        transactions++;
        return true;
    }

    void deposit(double amount)
    {
        balance += amount;
        transactions++;
    }

    double getBalance() const { return balance; }
    int getTransactions() const { return transactions; }
};

#endif

Question 1: Find the error in this class definition:

Question 11: Complete the copy constructor:

class Array {
private:
    int* data;
    int size;
public:
    Array(int s) : size(s) {
        data = new int[size];
    }
    
    // Complete the copy constructor
    Array(const Array& other) {
        // Your code here
    }
};
Type: Complete the Code

Completed Code:

Array(const Array& other) {
    size = other.size;
    data = new int[size];
    for(int i = 0; i < size; i++) {
        data[i] = other.data[i];
    }
}

Explanation: Copy constructor performs deep copy of the dynamic array to avoid shallow copy issues.

Question 17: Find the memory leak in this code:

class StringWrapper {

Question 20: Write a function to validate Rectangle dimensions:

        // Write validation logic here
        width = w;
        length = l;

Solution:

void setDimensions(double w, double l) {
    if (w > 0 && l > 0) {
        width = w;

        width = length = 1.0; // Default values

class BankAccount {
private:
    string accountNumber;
    double balance;
    string accountHolderName;

public:
    BankAccount(string accNum, string name, double initialBalance = 0.0) {
        accountNumber = accNum;
        accountHolderName = name;
        balance = initialBalance;
    }

    void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        }
    }

    bool withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            return true;
        }
        return false;
    }

    double getBalance() const { return balance; }

    void getAccountInfo() const {
        cout << "Account: " << accountNumber << endl;
        cout << "Holder: " << accountHolderName << endl;
        cout << "Balance: $" << balance << endl;
    }
};
